The Three Examples

The Three DeFi Primitives

N1 · Sealed RFQ

Request for Quote — simplest matching

  • 1 taker requests a price
  • N makers submit quotes via TLS
  • Enclave picks lowest price (argmin)
  • Walrus stores full quote set as proof
  • Solidity stub: token swap settlement

Move 94 LOC Rust 133 LOC Sol 151 LOC

N2 · Sealed Liquidation

Sealed-bid auction — different economics

  • Position becomes undercollateralized
  • N liquidators submit sealed bids
  • Enclave picks highest bonus (argmax on bps)
  • Walrus stores all bids as proof
  • Solidity stub: lending protocol liquidation

Move 101 LOC Rust 127 LOC Sol 162 LOC

N3 · Sealed Batch Swap

Crossing-order match — most complex

  • N buyers + M sellers submit intents
  • Sort buys desc, sells asc by limit price
  • Find clearing price at crossing point
  • Fill all orders within limits
  • Solidity stub: atomic batch settlement

Move 91 LOC Rust 175 LOC Sol 149 LOC

Shared Infrastructure

enclave.move

209 LOC · vendored from MystenLabs/nautilus

  • EnclaveConfig<T> — PCR storage
  • Enclave<T> — pubkey storage
  • verify_signature() — Ed25519 + BCS
  • IntentMessage<T> — signed payload wrapper

seal_bootstrap.rs

295 LOC · two-step Seal key load

  • init_seal_key_load — FetchKeyRequest
  • complete_seal_key_load — cache keys
  • encrypt_checkpoint — local IBE (no network)
  • decrypt_checkpoint — requires cached keys

walrus.rs

94 LOC · HTTP blob client

  • upload() — PUT to publisher
  • fetch() — GET from aggregator
  • Handles newlyCreated + alreadyCertified
  • Configurable endpoints + epoch count
