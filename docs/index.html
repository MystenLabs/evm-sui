Learning site · companion to the repo

Walrus behind your EVM contract, explained one showcase at a time.

The evm-nft-wal repo ships an NFT dApp plus five sibling showcases that swap fragile IPFS pieces for Walrus equivalents — without touching the EVM contract shape builders already know. This site is the guided tour: one page per showcase, pain point first, then the ~30–150 lines of code that fix it, then the exact command to run it locally.

01 · what this is

One repo, six tiny dApps, one shared thesis

Every showcase keeps the EVM contract exactly the shape builders already know — OpenZeppelin ERC-721, a plain registry mapping, a Snapshot-style proposal struct — and changes only two things: what your frontend uploads to and what URL prefix it reads from. The contract never learns about Walrus.

That symmetry is the whole point. If you can migrate one IPFS-backed dApp's URL prefix you can migrate them all, and the comparison between IPFS and Walrus becomes apples-to-apples instead of architectural hand-waving. Read showcase 01 first; the others are variations on the same template.

Same OpenZeppelin contract. Same browser fetch shape. One URL prefix flipped. That's the migration cost the repo is showing.

01 · shipped reference

EvmWal NFT — image + metadata on Walrus

Vanilla OpenZeppelin ERC-721 with a Walrus aggregator URL in tokenURI. Two write paths (operator SDK or public publisher), one read path, identical EVM surface to an IPFS-backed NFT.

shipped ~200 LOC migration: 1 URL prefix
Read the deep-dive 02 · frontend hosting

Walrus Sites — host the dApp itself on Walrus

One site-builder publish command, a SuiNS link, and the public wal.app portal resolves it for anyone. No IPNS, no Cloudflare gateway, no DNSLink fragility. Optional ENS bridge via CNAME or DNSLink.

scaffolded CLI + 1 HTML migration: SuiNS link
Read the deep-dive 03 · IPNS replacement

WalrusResolver — ENS-gated mutable pointer

A ~35-line Solidity contract: an ENS name owner parks a Walrus blob pointer on-chain. One eth_call resolves it (sub-200ms vs IPNS's DHT random walk). A permissionless keeper keeps the underlying blob alive forever.

implemented ~35 LOC sol + keeper migration: 1 contract deploy
Read the deep-dive 04 · DAO proposals

Governance — Snapshot-style proposals, body on Walrus

Proposer uploads the proposal markdown to Walrus (publisher pays WAL, not the DAO), contract stores just the 32-byte blobId + deadline + tallies. Replaces a 4Everland-pinned dependency with a content-addressed blob.

implemented ~100 LOC flash-loan warning
Read the deep-dive 05 · token lists

Verifiable manifest — Uniswap-shaped token list, on Walrus

Combine showcase 03's mutable pointer with a strict JSON schema. One eth_call resolves tokens.uniswap.eth to a Walrus blob; one aggregator GET returns the token list. The aggregator is the trust boundary — same as fetching by CID — but the pointer is deterministic instead of an IPNS lottery.

implemented ~80 LOC migration: ipfs:// → blobId
Read the deep-dive 06 · collection drops

QuiltedCollection — 10 000 tokens, one Walrus Quilt

Pack every image + metadata file into one Walrus Quilt with each token's filename as its identifier. The contract stores one quiltId; tokenURI(id) deterministically returns the aggregator URL. One upload, 10 000 retrievable blobs, no pin-management surface.

implemented ~120 LOC requires quilt CLI

04 · primitives glossary

The six Walrus and Sui pieces every showcase uses

If you've built on IPFS before, the cognitive mapping is small. Each primitive below replaces a specific IPFS concept — or fills a gap IPFS never had a clean answer for.

replaces · ipfs gateway

Walrus aggregator

A public HTTPS endpoint that serves a blob by its content address: /v1/blobs/<blobId>. Anonymous, no auth, no SDK. Used in every read path on this site.

replaces · pinning service

Walrus publisher

A public HTTPS endpoint that accepts PUT /v1/blobs?epochs=N with raw bytes and returns a blob id. Publisher pays WAL on the user's behalf — zero infra to upload.

new · no ipfs analogue

On-Sui Blob object

Every blob also exists as a Sui object whose end_epoch field encodes its retention. Operators can extend, delete, or transfer it on-chain — replacing fragile pinning-service state with on-chain truth.

new · scales pin counts

Walrus Quilt

A single blob that packs many files, each addressable by an identifier. Powers showcase 06: one upload, 10 000 retrievable patches at /v1/blobs/by-quilt-id/<quiltId>/<id>.

replaces · dnslink

SuiNS · wal.app portal

SuiNS is Sui-side naming; wal.app is the public portal that resolves a SuiNS name to a Walrus Site object. Powers showcase 02 — your dApp's hosting becomes a your-name.wal.app URL.

unchanged · ens still works

ENS contenthash · DNSLink

Your .eth name still works as your public identity. CNAME or DNSLink from there to wal.app bridges Walrus content into the existing web2 / ENS surface area users already trust.
