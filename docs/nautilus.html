When an EVM contract can't verify the off-chain part.

Showcases 01–06 swap the storage layer behind a Solidity contract. These swap the computation layer. A Rust enclave fetches external data, signs the result with an ephemeral ed25519 key; a Sui Move package verifies the signature. One pattern — IntentMessage + BCS + verify_signature — applied to twelve trust-failure categories where EVM developers keep hitting a wall.

Repo: showcases/07-nautilus/ Stack: Rust · axum · ed25519-dalek · Sui Move 2024 · BCS Upstream: MystenLabs/nautilus
the pattern

One Rust binary signs. One Move package verifies. BCS in the middle.

Every Nautilus showcase follows the same shape. The enclave (Rust + axum) exposes POST /process_data. It does the sensitive work — fetching a price, running ML inference, matching sealed bids — wraps the result in an IntentMessage<T>, BCS-encodes it, and signs with its ephemeral ed25519 key.

The Move package mirrors the payload struct field-for-field and calls verify_signature against the enclave's registered pubkey. Intent bytes separate domains so signatures from one app can't be replayed against another.

All examples run locally — deterministic signing key, all-zero PCRs, no AWS, no Marlin Oyster, free. The on-chain verifier is the same one that ships to production. 

// shared shape across all twelve categories

// Rust: sign the payload
let resp = MyPayload { field_a, field_b, ... };
return to_signed_response(&kp, resp, ts, MY_INTENT);

// Move: verify the same BCS bytes
let payload = MyPayload { field_a, field_b, ... };
assert!(enclave.verify_signature(
    MY_INTENT, ts, payload, sig
), EInvalidSignature);

The trust failure

field report

Honest contrast — where Nautilus wins and where it doesn't.

Wins outright

  • Greenfield Sui apps that need verifiable off-chain compute.
  • App-specific oracles (social, sports, niche feeds) — cleaner than Chainlink Functions' BETA path.
  • Confidential matching engines on Sui with public settlement.
  • AI agents whose code identity matters: PCRs commit to which binary holds the key.
  • Reproducible-build audit story — the PCR is a function of the source.

Wrong tool for the job

  • Must remain on EVM for liquidity — use Phala, Marlin Oyster, Oasis Sapphire.
  • Fully confidential EVM by default — Oasis Sapphire is purpose-built.
  • MEV infrastructure for Ethereum — Flashbots SUAVE owns this lane.
  • Multi-vendor TEE diversity — Phala's roadmap is broader.
  • Pure VRF or timestamping — Sui has native primitives; Nautilus is overkill.
